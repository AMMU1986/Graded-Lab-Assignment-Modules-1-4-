# Question 5: Storage Health Assessment

# 1. Available storage devices
lsblk
fdisk -l

# Explanation: lsblk lists block devices in a tree format, fdisk -l shows detailed partition information.

# 2. Mounted file systems
df -h
mount | column -t

# Explanation: df -h shows disk usage with human-readable sizes, mount displays mounted filesystems.

# 3. Disk usage statistics
du -sh /*
du -sh /home/*

# Explanation: du -sh shows size summary of directories, /* checks root, /home/* checks user directories.

# 4. Inode utilization
df -i
df -i /dev/sda1

# Explanation: df -i shows inode usage statistics, helps identify inode exhaustion issues.

# 5. Detailed filesystem info
tune2fs -l /dev/sda1

# Explanation: Displays filesystem parameters, features, and last check times.

# 6. SMART data (if available)
sudo smartctl -a /dev/sda

# Explanation: SMART data provides disk health information (requires sudo).

# 7. I/O statistics
iostat -x 1 3

# Explanation: Shows disk I/O statistics including wait time, service time, and utilization.

# 8. Storage performance test
dd if=/dev/zero of=./testfile bs=1M count=100 conv=fdatasync

# Explanation: Creates a 100MB file to test write speed.

# 9. Create final report using vi (on your system)
vi Storage_Assessment_Report.txt
